PAK.sh Commands

Available commands and their usage for the PAK.sh package automation system.

Core Commands

Installation

pak install <package>

Install a package with automatic dependency resolution

pak install --file packages.txt

Install multiple packages from a file

Management

pak list

List all installed packages

pak update <package>

Update a specific package

pak remove <package>

Remove a package and its dependencies

Information

pak info <package>

Show detailed information about a package

pak search <query>

Search for packages

System

pak init

Initialize PAK.sh in the current directory

pak config

Show or edit configuration

pak doctor

Check system health and diagnose issues

Advanced Commands

Batch Operations

pak batch install packages.txt

Install multiple packages in parallel

Rollback

pak rollback <package> --version <version>

Rollback a package to a specific version

Telemetry

pak telemetry enable

Enable telemetry data collection

pak telemetry disable

Disable telemetry data collection

Configuration

PAK.sh can be configured using environment variables or a configuration file:

Environment Variables

  • PAK_TELEMETRY_URL - Telemetry endpoint URL
  • PAK_API_KEY - API key for telemetry
  • PAK_LOG_LEVEL - Logging level (debug, info, warn, error)
  • PAK_CACHE_DIR - Cache directory path

Configuration File

Create a pak.config file in your project root:

[telemetry]
enabled = true
url = https://yourdomain.com/api/telemetry
api_key = YOUR_API_KEY_HERE

[logging]
level = info
file = pak.log

[cache]
directory = .pak/cache
max_size = 1GB
